Block(二)：Blcok的使用

为常用的Blcok创建typedef

为了隐藏复杂的块类型，需要用到C语言的类型定义的特性：

typedef int (^NewBlockType)(BOOL flag, int value);

blockType就被定义为一种需要BOOL和int两个参数，返回值为int型的新Blcok对象类型。

NewBlockType block = ^(BOOL flag, int value) {
    //Block实现
}；
block(YES, 1);

使用类型定义还有个好处，就是你打算重构Block的类型签名的时候会很方便。

//
//  NetWorkHandler.h
//  Block 
//  Created by Paranorman on 2017/1/18.
//  Copyright © 2017年 Paranorman. All rights reserved.

#import <Foundation/Foundation.h>

typedef void(^ PNetWorkCompletionHandlerWithError)(NSData *data, NSError *error);

@interface NetWorkHandler : NSObject

@property(nonatomic, strong) NSString *url;

1.-(void)startWithCompletionHandler: (void(^)(NSData *data)) completion;
2.+(void)startWithCompletionHandlerError:(PNetWorkCompletionHandlerWithError) completion;

@end

例如上面的.h文件 你里面有很多个带Block同类型参数的方法。当你需要重新构造该Block类型对象时候。1方法没有使用typedef，因此你需要在每一个方法名称上修改Block类型参数。但是你要是使用了typedef，你仅仅需要在类型定义的时候去修改Blcok类型参数。如果这个类有多个Blcok类型，这样会显得很方便。

用handler块降低代码分散程度

当我们在写一些回调功能的时候，常常会用到delegate设计模式。但是代理设计模式的代码分散程度太高，换做Block回调就会简洁的多。

假设上面的头文件就是我们的网络处理器，那么在.m文件中的实现大体如下:

- (void)startWithCompletionHandler:(void (^)(NSData *))completion {
NSLog(@"startWithCompletionHandler");
if (completion) {
    completion(nil);
}}

那么在使用这个网络请求对象的类里面处理业务逻辑就可以直接在Block的方法体内实现，这样比delegate看上去简洁的多。当然使用Block时候要注意避免构成保留环。
NetWorkHandler netHandler = [[NetWorkHandler alloc] init];
[netHandler startWithCompletionHandler:^(NSData data) {
NSLog(@”%@”, netHandler.url);
//
}];

利用Block实现链式开发

第三方框架Masonry在实现一个自动布局时候的实现代码:

[self.demoTextField mas_makeConstraints:^(MASConstraintMaker *make) {
    make.top.equalTo(self.view).and.offset(100);
    make.width.equalTo(self.view);
    make.height.equalTo(@44);
}];

其内部的设计思想就是利用oc的点语法来返回一个self，来继续调用自身的实例方法。

 #import <Foundation/Foundation.h>
@class Person;
typedef Person *(^PersonBlcokType)(int i);

@interface Person : NSObject

-(PersonBlcokType)eat;
-(PersonBlcokType)sleep;
-(PersonBlcokType)dadoudou;

@end

@implementation Person

- (PersonBlcokType)eat {
return ^ Person *(int i) { NSLog(@"eat"); return self;};
}

- (PersonBlcokType)sleep {
return ^ Person *(int i) { NSLog(@"sleep"); return self;};
}

- (PersonBlcokType)dadoudou {
return ^ Person *(int i) { NSLog(@"dadoudou"); return self;};
}

在外部调用这个类会是这个样子：

Person *p = [[Person alloc] init];
p.eat(1).sleep(1).dadoudou(1);

利用Blcok返回自身对象来实现链式，利用Block的参数传递来修改类的一些数据属性，实现函数式。这就是Masonry内部的利用Block的例子。

Block(一)：语法与基本使用

Block概要

块（Block）是Objective-c对ANSI C做的扩展：带有自动变量（局部变量）的匿名函数。

我们先来看看C语言下的函数：

int func(int count);

int result = func(1);

C语言不允许使用匿名函数，而且调用函数的时候也需要用函数名func来调用函数。

再来看看函数指针的调用情况:

int result = (*funcPtr)(1);

看上去确实不需要函数名称便可以调用fun函数，实际上在定义函数指针的时候:

int (*funcPtr)(int) = &func ;

int result = (*funcPtr)(1);

从代码上看，实际上还是需要函数名称作依托。那么我们接下来就看看Block这种匿名函数是如何使用的。

Block模式

Block语法

定义块（Block）的语法格式如下：

^返回值类型（形参类型1 形参1，形参类型2 形参2，...） {

}

与C语言的函数有两点不同：

1.没有函数名。
2.带有^.

第一点不同是因为Blcok是匿名函数，所以没有函数名称。第二点不同是返回值类型前面带有“^”记号，插入记号便于查找。

Blcok省略形式

1.省略返回值类型：

^（形参类型1 形参1，形参类型2 形参2，...） {

}

省略返回值类型时候，如果表达式语句中有return语句就使用该返回值类型，如果没有return语句就是用void累心。表达式中含有多个return语句时，所有return的返回值类型必须相同。

^(int count) {return count + 1}

该BLock语法讲按照return语句的类型，返回int型返回值。

2.省略参数类型：

^ {

}

如果不使用参数，参数列表也可以省略。

^{NSlog(@"Block");}

Blcok类型变量

在C语言的语法中，可以将所定义的函数的地址赋值给函数指针类型的变量中，通过函数指针就可以调用所指向的函数。

int func （int count） {
    return count + 1；    
}
int （*funcPtr）（int） = &func；
int result = （* funcPtr)(1)；

同样地，在Blcok中可以将Block语法生命的Block对象赋值给Blcok类型的变量中。

Block类型的变量如下：

返回值类型（^Block变量名)(形参类型1，形参类型2，...)

1.定义无参数无返回值的Blcok变量：

void (^blk)(void) = ^void (void) {};

2.定义有参数无返回值的Block变量
void (^blk)(int) = ^ void (int count) {};

3.定义无参数有返回值的Blcok变量
int (^blk)(void) = ^int (void) {return 1;}

4.定义有参数有返回值的Blcok变量
int (^blk)(int) = ^int (int count) {return count + 1;};

Blcok调用

调用Block与函数指针的调用几乎完全相同。注：以下调用与上面Block变量对应。

blk();

blk(1);

int result = blk();

int result = blk(1);

Block类型的变量完全可以向C语言变量一样使用，也可以使用指向Block类型变量的指针，即Block的指针类型变量。

typedef int (blk_t)(int);

blk_t blk = ^(int count){return count + 1;};

blk_t *blkPtr = &blk;

int result = (*blkPtr)(1);

截取自动变量值（局部变量）

我们已经知道了Blcok的匿名函数的意思，“带有自动变量”代表“截获自动变量值”。
看一下下面的代码:

int val = 10;
void ( ^blk)(void) = ^{
    NSLog(@"val = %d", val);
};
val = 5;
blk();

输出：

2017-01-18 10:43:14.546 Block[999:38471] val = 5

Block语法表达式在使用局部变量val时，在定义块的时候就会把局部变量的值保存到块中，而不是等到执行的时候才去访问，因此后面对局部变量val的任何修改都不会对Blcok产生印象，我们称之为”截获自动变量”。并且在Block内部不能修改局部变量的val的值，即：将值赋值给在Block中截获的自动变量，就会产生编译错误。

如果你不希望在定义块的时候吧局部变量的值复制到Blcok中，而是等到执行时候才去访问，甚至在Block内部改变局部变量的值，此时可以考虑使用__block修饰局部变量。

__block int val = 10;
__block int tmp = 10;
void ( ^blk)(void) = ^{
    val = 100;
    NSLog(@"\nval = %d \ntmp = %d", val, tmp);
};
tmp = 5;
blk();

输出：

2017-01-18 10:53:26.029 Block[1245:44613] 
val = 100 
 tmp = 5

截取OC可变对象

截取oc可变数组，调用变更该对象的方法，如下面代码所示，这样是允许的。

NSMutableArray *array = [NSMutableArray array];
void ( ^blk)(void) = ^{
    [array addObject:@1];
};
blk();

截取C语言数组

const char text[] = "hello world";
void ( ^blk)(void) = ^{
    NSLog(@" %c", text[1]);
};
blk();

这样在Block内部调用C语言数组，编译器会报错。这是因为在Block内，截取自动变量的方法并没有实现对C原因数组的截获。这时，可以使用指针解决该问题。

const char *text= "hello world";
void ( ^blk)(void) = ^{
    NSLog(@" %c", text[1]);
};
blk();

全局块、栈块及堆块

定义块的时候，其所占的内存区域是分配在栈中的。这就是说，块只在定义它的那个范围内有效。（ARC下编译器大多数情况下会判断帮助我们做了copy操作）
例如：Block作为函数返回值时，编译器认为你这个Block出了函数作用域依然有效，就会自动帮我们对Block对象做copy操作，保证Block从栈内存复制到堆内存上。但是，你的函数要是返回一个Blcok对象的数组，编译器就不知道是不是要帮你做这次Copy操作，这就需要我们自己copyBlcok对象。

-（Nsarry *）getBlcokArray {
int val = 10；
return [NSArray arrayWithObjects:
            [^{NSlog(@"%d", val);}copy],
            [^{NSlog(@"%d", val);}copy],nil];
}
(调用局部变量val保证Blcok不是全局的存储在静态数据区)

为了解决此问题，可给块对象发送copy消息拷贝。这样就可以把块从栈内存复制到堆内存。而且，一旦复制到堆上，块就成了带引用计数的对象了。

除了栈Blcok（MRC），堆Blcok，还有一种全局Blcok：
1.定义在函数外面的block是global类型的
2.定义在函数内部的block，但是没有捕获任何自动变量(包括oc对象)，那么它也是 全局的

iOS10 新openURL的特性

openURL:

在iOS10之前，我们常使用openURL：来实现一些在应用内打开一些外部链接的需求。并且利用canOpenURL：来进行跳转的判断。

iOS10

iOS出现了新的替代方法openURL:options:completionHandler:

Apple官方文档描述如下：

 The new UIApplication method openURL:options:completionHandler:, which is executed asynchronously and 
calls the specified completion handler on the main queue (this method replaces openURL:).

简单理解为：异步执行openURL操作和主线程执行回调判断跳转success.

option：

options目前可传入参数Key在UIApplication头文件只有一个:UIApplicationOpenURLOptionUniversalLinksOnly,其对应的Value为布尔值,默认为False.若传入YES则代表只能通过该URL应用打开，并不能通过Safari打开URL链接

例子:

[[UIApplication sharedApplication] openURL:[NSURL URLWithString:@"http://www.baidu.com"] options:@{} completionHandler:^(BOOL success) { }];

使用hexo+GitHub搭建博客

注：我使用的macOS Sierra， Windows下基本无差别。

1.GitHub账号

创建仓库

点击New repository效果如下图


注意：你的repository名字要与你的GitHub账号名字相同，有可能会出现404的显示结果。

生成添加秘钥

在终端输入：

ssh -keygen -t rsa -C "GitHub的注册邮箱"

enter之后会在你的目录下生成两个文件：xxx_rsa xxx_rsa.pub(xxx名字是用户自己在终端输入生成的)。用mac自带记事本打开.pub文件，复制里面内容。
找到GitHub的Personal settings的下的 SSH and GPG keys选项。
入口:
https://github.com/settings/keys
讲复制内容粘贴到SSH Key的输入框，title的内容随意，点击

Add SSH key

即可。

2.安装node.js

按照官网下载安装即可。

3.安装git

由于我的电脑装有Xcode，自带Git，故和我一样的xcode的mac用户请跳过此步骤。
其他用户点击这里下载安装git，如下图。

4.安装配置hexo

因为hexo是一款基于node.js的静态博客框架，前面安装的node.js就是为它准备的，它也是我们搭建博客的关键。

(1)打开终端，进入博客的本地放置路径，输入如下指令：

$ npm install -g hexo-cli

若提示error，请输入：

$ sudo npm install -g hexo-cli

按照命令号安装,安装好hexo，在终端输入:

$ hexo

终端显示如下，安装成功：

(2)初始化博客配置文件

a.在你的博客目录下创建文件夹，初始化。

$ hexo init xxx

b.进入博客文件夹。

$ cd xxx

c.node.js命令，安装博客所需要的依赖包。

$ nmp install

初始化成功后，文件夹结构如下图所示：

(3)配置博客信息

找到_config.yml文件。

1.修改网站相关信息

title: paranorman
subtitle: 
description: 
author: paranorman
language: zh-CN
timezone: Asia/Shanghai

注意：每一项的填写:后面都要保留一个空格，下同。

2.配置个人域名

url ：http:Paranorman-Reborn-ACL.github.io.com

对于root（根目录）、permalink（永久链接）、permalink_defaults（默认永久链接）等其他信息保持默认。

3.配置部署。

deploy
    type: git
   repo: https://github.com/Paranorman-Reborn-ACL/Paranorman-Reborn-ACL.github.io.com
      branch: master

其中repo项是之前Github上创建好的仓库的地址，可以通过如下图所示的方式得到：

branch是项目的分支，我们默认用主分支master。

4.发表一篇博客

$ hexo new "title"

我们可以在本地博客文件夹source->_post文件夹下看到我们新建的markdown文件。

5.本地发布

$ hexo server

成功后下是如下：

浏览器输入url就可以看见我们的博客。

6.博客部署到GitHub

$ hexo deploy

如提示如下错误:

ERROR Deployer not found: git

输入如下指令：

npm install hexo-deployer-git --save

重新部署。

在浏览器输入GitHub部署网址即可，之前仓库的后缀名：yourname.github.io。
